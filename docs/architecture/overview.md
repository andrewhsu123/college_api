# 系统架构概览

## 整体架构设计

### 核心设计思路

- **角色管辖模型**：角色通过 JSON 字段存储管辖的机构和人员，用户通过角色获得管辖权限
- **核心查询场景**：快速查询"我管辖了哪些人员"（用户 → 角色 → 机构/人员）
- **大表优化**：persons 表缓存 + ES 搜索 + 索引优化
- **服务中台化**：统一人员、机构、权限服务，支持多业务系统接入

### 系统分层

```
┌─────────────────────────────────────────┐
│         业务应用层（各业务系统）          │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│      人员中心服务中台（API Gateway）      │
└─────────────────────────────────────────┘
                    ↓
┌──────────┬──────────┬──────────┬────────┐
│ 人员服务  │ 机构服务  │ 角色服务  │ 权限服务│
└──────────┴──────────┴──────────┴────────┘
                    ↓
┌──────────┬──────────┬──────────┬────────┐
│  MySQL   │  Redis   │    ES    │  MQ    │
└──────────┴──────────┴──────────┴────────┘
```

### 数据流向

```
用户登录 → 获取用户角色(persons_has_roles) 
       → 获取角色管辖范围(persons_roles.department_ids/person_ids)
       → 根据机构查询人员(students/staff) 
       → 返回管辖人员列表
```

## 服务职责划分

### PersonService（人员服务）
- 人员 CRUD
- 人员搜索（调用 SearchService）
- 人员角色管理（调用 RoleService）
- 学生/政工管理

### DepartmentService（机构服务）
- 机构 CRUD
- 机构树查询
- 机构人员查询（调用 PersonService）

### RoleService（角色服务）
- 角色 CRUD
- 角色管辖范围管理
- 角色成员管理

### PermissionService（权限服务）
- 权限计算（用户 → 角色 → 管辖范围）
- 权限校验
- 管辖人员/机构查询

### SearchService（搜索服务）
- ElasticSearch 索引管理
- 全文搜索
- 复杂条件查询

## 技术栈

- **数据库**：MySQL 5.7+（主从复制）
- **缓存**：Redis（集群）
- **搜索**：ElasticSearch（集群）
- **消息队列**：RabbitMQ / Kafka
- **开发语言**：PHP / Java / Go（根据团队技术栈选择）
