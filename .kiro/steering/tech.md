# 技术规范

## 技术栈

### 后端
- **语言**：Go 1.21+
- **框架**：Gin（HTTP路由）
- **数据库**：MySQL 8.0+
- **缓存**：Redis 7.0+
- **搜索引擎**：ElasticSearch 8.0+
- **消息队列**：RabbitMQ 或 Kafka

### 开发工具
- **依赖管理**：Go Modules
- **API文档**：Swagger/OpenAPI
- **代码规范**：golangci-lint
- **测试框架**：testing + testify

## 编码规范

### Go代码风格
- 遵循 [Effective Go](https://go.dev/doc/effective_go) 指南
- 使用 `gofmt` 格式化代码
- 包名使用小写单数形式
- 接口名使用 `-er` 后缀（如 `Reader`、`Writer`）
- 错误处理：显式检查每个错误，不要使用 `panic`

### 命名约定
- **变量**：驼峰命名法（camelCase）
- **常量**：大写驼峰命名法（PascalCase）或全大写下划线分隔
- **函数/方法**：大写驼峰命名法（导出）或小写驼峰命名法（私有）
- **文件名**：小写下划线分隔（snake_case）

### 项目结构
```
.
├── cmd/                    # 应用程序入口
├── internal/              # 私有应用代码
│   ├── handler/          # HTTP处理器
│   ├── service/          # 业务逻辑
│   ├── repository/       # 数据访问层
│   ├── model/            # 数据模型
│   └── middleware/       # 中间件
├── pkg/                   # 可复用的公共库
├── config/                # 配置文件
├── docs/                  # 文档
├── sql/                   # 数据库脚本
└── test/                  # 测试文件
```

## 数据库规范

### 表设计
- 使用 `InnoDB` 存储引擎
- 主键使用 `BIGINT UNSIGNED AUTO_INCREMENT`
- 时间字段使用 `DATETIME` 类型
- 必须包含 `created_at` 和 `updated_at` 字段
- 软删除使用 `deleted_at` 字段（可为NULL）

### 索引策略
- 为外键创建索引
- 为频繁查询的字段创建索引
- 复合索引遵循最左前缀原则
- 避免过多索引影响写入性能

### 查询优化
- 避免 `SELECT *`，明确指定需要的字段
- 使用 `LIMIT` 限制结果集大小
- 复杂查询使用 `EXPLAIN` 分析执行计划
- 大表查询考虑分页和缓存

## API设计规范

### RESTful原则
- 使用标准HTTP方法：GET、POST、PUT、DELETE
- URL使用名词复数形式：`/api/v1/persons`
- 使用HTTP状态码表示结果：200、201、400、404、500等
- 版本控制：在URL中包含版本号 `/api/v1/`

### 请求/响应格式
```json
// 成功响应
{
  "code": 0,
  "message": "success",
  "data": { ... }
}

// 错误响应
{
  "code": 1001,
  "message": "参数错误",
  "error": "详细错误信息"
}
```

### 分页格式
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [...],
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```

## 缓存策略

### 缓存层级
1. **本地缓存**：热点数据（如部门树）
2. **Redis缓存**：用户会话、权限数据、查询结果
3. **数据库**：持久化存储

### 缓存键命名
- 使用冒号分隔：`service:entity:id`
- 示例：`person:detail:123`、`dept:tree:all`

### 缓存过期
- 详情数据：30分钟
- 列表数据：5分钟
- 权限数据：15分钟
- 部门树：1小时

## 日志规范

### 日志级别
- **DEBUG**：详细的调试信息
- **INFO**：一般信息（请求日志、业务操作）
- **WARN**：警告信息（可恢复的错误）
- **ERROR**：错误信息（需要关注的异常）
- **FATAL**：致命错误（导致程序退出）

### 日志格式
```
[时间] [级别] [服务名] [trace_id] 消息内容
```

## 测试规范

### 单元测试
- 测试文件命名：`*_test.go`
- 测试函数命名：`TestXxx`
- 覆盖率目标：核心业务逻辑 > 80%

### 集成测试
- 使用测试数据库
- 测试前清理数据
- 测试后恢复环境

## 安全规范

### 认证授权
- 使用JWT进行身份认证
- Token过期时间：2小时
- 刷新Token机制

### 数据安全
- 敏感信息加密存储
- SQL注入防护：使用参数化查询
- XSS防护：输出转义
- CSRF防护：使用Token验证

### 接口安全
- 接口限流：防止恶意请求
- 参数校验：严格验证输入
- 日志脱敏：不记录敏感信息

## 性能优化

### 数据库优化
- 使用连接池
- 批量操作代替循环单条
- 读写分离
- 分库分表（必要时）

### 缓存优化
- 缓存预热
- 缓存穿透防护：布隆过滤器
- 缓存击穿防护：互斥锁
- 缓存雪崩防护：过期时间随机化

### 代码优化
- 避免N+1查询
- 使用goroutine处理并发任务
- 合理使用sync.Pool复用对象
- 避免过度使用反射
